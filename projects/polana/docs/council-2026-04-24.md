## Game Designer

**Режь scope немедленно:**
- 5 фракций в демо → 3 максимум (КБ, КС, БГ)
- Фоновые NPC (дед, мама, велосипедист) → в беклог
- PvP → только после демо

**Критические дыры в механиках:**
- Отморозок + КБ = сломанная комбо. Friendly fire → реген брони работает как бафф для тех, кто бьёт своих. Ограничить: КБ-бонус не триггерится от friendly fire, либо Отморозок получает malus = КБ-регену.
- Молодой с 3×/провал — не работает в autochess. Монета на ядре юнита = фрустрация. Заменить на +50% эффект при успехе / -20% при провале.
- Хардкор и Выездной — один архетип (оба баффер). Развести: Выездной = активная способность (щит/хил/cleanse), Хардкор = пассивная аура + сам танк.
- Все 5 фракций дают «бей сильнее» — нет стилевого разнообразия. Нужны механически уникальные идентичности: контроль, мобильность, эко-скейлинг.

**Критически отсутствует в плане:**
- Экономика (золото, интерес, реролл)
- Размер борда и лимит юнитов по уровню
- Способности юнитов (ни одной расписано)
- Предметы
- Win/loss condition раунда (HP игрока, формула урона)

**Про банды:** названия банд — флавор в тултипе. Основной UI сигнал силы юнита — цифра/звезда (1-2-3), не название. 45 названий игрок не считает за 0.3 сек.

## Tech Architect

**Главные риски форка ChraumaTactics:**
- Augment-система ≠ фракции+архетипы. Нужно переписать SynergyManager. 1–2 недели.
- TABS ragdoll — самый недооценённый риск. ChraumaTactics на Mecanim, TABS — active ragdoll. Решение: ragdoll только на смерти (not active). Для «TABS-вайба» хватит.
- Проверить лицензию форка до первого коммита. GPL = проблемы на Steam.
- Мультиплеер мешает PvE-разработке каждый день. Запарковать в ветку `multiplayer-parked` сразу.

**Правильный порядок разработки (5–6 недель):**
1. Неделя 1: форк + апгрейд Unity 6 + починка + ScriptableObjects данных + EventBus + выбор одного MCP
2. Неделя 2: MergeManager + 3 фракции с простыми бонусами + 9 юнитов-капсул
3. Неделя 3: 3 архетипа как Strategy-компоненты (не наследование) + автобой-балансировка
4. Неделя 4: простой бот + полный игровой цикл (shop→prep→combat→результат) + UI банд
5. Неделя 5–6: low-poly модели + ragdoll на смерть + плейтест + фиксы

**Про MCP:** выбрать один сразу. Агенты хорошо пишут изолированные менеджеры и ScriptableObjects, но не настраивают Animator и ragdoll — это руками.

**Вердикт: playable демо за 5–6 недель при правках. Без правок — 3–4 месяца.**

## Monetization / Biz

**Жанровый риск:**
- Autochess на PC в 2026 — умирающий жанр. Underlords закрыта. TFT жив только внутри LoL.
- Нишевая тема × нишевый жанр = малые числа. Потолок 20–50k продаж при отличном исполнении.

**Юридические риски:**
- Stone Island, Adidas, Fred Perry, Burberry — убрать. Заменить на fictional бренды с узнаваемыми отсылками (как GTA: Pißwasser вместо Budweiser).
- Цвета клубов без логотипов/имён — серая зона, рабочая.
- Российское законодательство: «романтизация насилия группами лиц» — риск. Нужен юрист перед РФ-релизом.
- Консоли (PS/Xbox/Switch) — не рассматривать, зарубят на сертификации.

**Рекомендуемая модель: Premium $12–15 + косметика**
- НЕ F2P (не потянешь live-service)
- Косметические паки по фракциям ($3–5): скины банд, анимации победы
- DLC-фракции (LP, ЧБ и далее) ($5–7)
- Steam Workshop для пользовательских банд
- Саундтрек как DLC

**Критично:** нужен PvE / roguelike режим. Без него при провале PvP игра = ноль. Если нет готовности делать PvE — закрыть на стадии демо.

## Консенсус (все трое согласны)

1. Scope демо: 3 фракции, 3 архетипа, ~9 юнитов
2. Мультиплеер — в беклог, не мешать демо
3. Визуал — последний приоритет, капсулы с цветом фракции достаточно для старта
4. Реальные бренды — заменить на fictional до релиза
5. PvE-режим обязателен как страховка от провала PvP
